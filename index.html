MyBatis随笔

什么是MyBatis?

MyBatis 是一款优秀的持久层框架，针对JDBC技术进行了封装。MyBatis 避免了几乎所有的 JDBC 代码和手动设置参数以及获取结果集。MyBatis 可以使用简单的 XML 或注解来配置和映射原生类型、接口和 Java 的 POJO（Plain Old Java Objects，普通老式 Java 对象）为数据库中的记录

为什么要学习MyBatis?

  • JDBC技术进行一个对比
  • JDBC代码冗余，除了sql命令不一样，其他地方基本相同
  • JDBC中的查询方法，对于结果集的数据封装，需要手动创建实体类对象
  • JDBC技术自身没有提供 “连接池” 功能
  • JDBC技术没有提供 “缓存” 功能

MyBatis框架是一个基于持久层的框架，是对我们现有代码JDBC的合理封装，解决的是我们开发过程中数据库持久层（DAO）的问题，MyBatis为我们提供了一种全新的机制来替换我们原有的jdbc功能

##MyBatis开发步骤：

  • 搭建开发环境
    • 导入jar包
      • ojdbce6.jar(JDBC连接数据库所需的驱动jar)
      • mybatis.jar(mybatis所需的核心jar)
      • log4j.jar(打印输出日志文件)
    • 引入配置文件
      • mybatis-config.xml( mybatis的核心配置文件)
      • mapper.xml(mapper映射文件)
      • log4j.properties(打印日志的相关配置文件)
    • 初始化配置
      • mybatis-config.xml(配置当前数据库运行环境)
  • 建表
  • 创建实体化对象
  • 定义DAO接口
  • 配置Mapper映射文件(取代的是DAO接口的实现类)
  • 注册Mapper映射文件
  • 获取Mapper映射文件所提供的实现类对象(工具类SqlSession)

    ###配置SSH key
    我对SSH key的理解就是通过SSH协议让本地的电脑与你GitHub库取得连接，可以在本地远程管理库中的代码，而且提高了安全性

打开Git bash窗口，输入：

ssh-keygen -t rsa -C "邮箱地址"

然后连续按三次回车，最终会在用户目录下生成.ssh文件夹，进入其中，用文本编辑器打开id_rsa.pub文件，复制全部内容，打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key，将刚复制的内容粘贴到key那里，title随便填

测试是否成功

输入以下代码：

$ ssh -T git@github.com # 注意邮箱地址不用改

根据提示Are you sure you want to continue connecting (yes/no)?输入yes
如果出现Hi xxx! You've successfully authenticated, but GitHub does not provide shell access.即成功
